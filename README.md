## 🧱 Architecture & Code Style

- **Small services** with clear boundaries; prefer composition over inheritance
- **Async-first** APIs, cancellation tokens where it matters
- **Errors as data**: structured logs + problem details for HTTP
- **Testing pyramid**: fast unit tests, a few integration tests, rare E2E
- **CI first**: format, lint, test on every push; build artifacts for releases
- **Documentation**: README, make targets, and example requests
